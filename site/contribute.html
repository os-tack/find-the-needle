Contribute a Benchmark

What makes a good benchmark?

  • A real bug from a real codebase — not synthetic, not contrived
  • A Dockerfile that builds the broken environment in under 5 minutes
  • A test.sh that fails on the bug and passes on the fix
  • A sealed .bench/solution.patch the agent never sees

Directory structure

benchmarks/<name>/
  Dockerfile
  Agentfile
  test.sh
  README.md
  .bench/
    solution.patch

Getting started

  1. Read the full CONTRIBUTING.md
  2. Copy the benchmarks/_template/ directory
  3. Package your bug, write a test, generate a patch
  4. Open a PR — CI will validate everything automatically
